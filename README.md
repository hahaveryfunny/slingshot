## ‚öñÔ∏è Design & Architecture Trade-offs

### Singleton-Based Architecture
**Why it was used**
- Simpler mental model for a first full game
- Faster iteration during early development
- Easy global access to core systems

**Trade-offs**
- Tight coupling between systems
- Harder to test and extend cleanly
- Less flexible than event-driven or decoupled architectures

Later projects in this portfolio move away from this approach in favor of **event-based communication and modular systems**, reflecting an evolution in design practices.
